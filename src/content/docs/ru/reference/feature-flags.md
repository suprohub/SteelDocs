SteelMC использует флаги функций Rust (растовские --features) для включения или отключения функциональности на этапе компиляции. Используйте их, чтобы адаптировать сборку для продакшена, отладки, профилирования, телеметрии или обслуживания сгенерированного кода.

Большинство пользователей собирают серверный бинарный файл через пакет `steel`:

```bash
cargo build -p steel --features "название_функции"
```

## steel

Эти флаги управляют основным исполняемым файлом сервера.

| Флаг | По умолчанию | Описание |
|---------|---------|-------------|
| `mimalloc` | Да | Использует MiMalloc в качестве глобального аллокатора для повышения производительности выделения памяти. |
| `stand-alone` | Нет | Встраивает иконку сервера по умолчанию и записывает её, если указанный файл иконки отсутствует. |
| `deadlock_detection` | Нет | Включает обнаружение взаимных блокировок `parking_lot` для отладки. |
| `dhat-heap` | Нет | Включает профилировщик кучи DHAT для анализа памяти. При включении заменяет MiMalloc в качестве глобального аллокатора. |
| `spawn_chunk_display` | Нет | Показывает цветную сетку в терминале во время генерации стартовых чанков. |
| `slow_chunk_gen` | Нет | Замедляет генерацию чанков при подготовке спавна и пробрасывает `steel-core/slow_chunk_gen`; полезно для наблюдения за отображением стартовых чанков. |
| `jaeger` | Нет | Включает экспорт трассировки OpenTelemetry через OTLP для Jaeger-совместимых систем трассировки. |

### Примеры использования

```bash
# Сборка с обнаружением взаимных блокировок
cargo build -p steel --features deadlock_detection

# Сборка без стандартного аллокатора MiMalloc
cargo build -p steel --no-default-features

# Сборка с отображением стартовых чанков
cargo build -p steel --features "spawn_chunk_display slow_chunk_gen"

# Сборка с поддержкой трассировки Jaeger/OpenTelemetry
cargo build -p steel --features jaeger
```

## steel-core

Эти флаги управляют основной игровой логикой и низкоуровневыми отладочными помощниками.

| Флаг | По умолчанию | Описание |
|---------|---------|-------------|
| `stand-alone` | Нет | Определён в манифесте крейта, но в настоящее время не имеет активных путей кода `cfg(feature = "stand-alone")` в `steel-core`. Флаг `stand-alone` серверного бинарного файла отвечает за поведение встроенной иконки. |
| `slow_chunk_gen` | Нет | Добавляет небольшую задержку после этапов генерации чанков, когда сервер включает медленную генерацию спавна. |
| `flint` | Нет | Предоставляет вспомогательные средства для интроспекции поведений, включая конкретные имена типов поведений и срезы реестра поведений. |

## steel-registry

Эти флаги управляют системой игровых реестров и сгенерированным кодом реестров.

| Флаг | По умолчанию | Описание |
|---------|---------|-------------|
| `fmt` | Нет | Запускает `rustfmt` для сгенерированных файлов реестров во время скрипта сборки. |
| `minecraft-src` | Нет | Включает помощники, которые сравнивают сгенерированные данные реестров с эталонными данными `minecraft-src`. |

## steel-utils

Эти флаги управляют сгенерированными утилитарными данными.

| Флаг | По умолчанию | Описание |
|---------|---------|-------------|
| `fmt` | Нет | Запускает `rustfmt` для сгенерированных утилитарных файлов во время скрипта сборки. |

## steel-worldgen

Эти флаги управляют сгенерированными данными генерации мира.

| Флаг | По умолчанию | Описание |
|---------|---------|-------------|
| `fmt` | Нет | Запускает `rustfmt` для сгенерированных файлов генерации мира во время скрипта сборки. |

## Комбинирование флагов

Можно включить несколько флагов в одной сборке:

```bash
# Сборка с несколькими отладочными функциями сервера
cargo build -p steel --features "deadlock_detection spawn_chunk_display slow_chunk_gen"

# Сборка для автономного распространения сервера
cargo build -p steel --release --features stand-alone

# Форматирование сгенерированного вывода реестра при сборке этого крейта
cargo build -p steel-registry --features fmt
```

Флаги функций имеют область видимости пакета. При включении функций вне пакета `steel` явно указывайте целевой пакет с помощью `-p`.

## Рекомендации для продакшена

Для продакшен-серверов рекомендуется:

- Оставлять `mimalloc` включённым, если только не профилируете с `dhat-heap`.
- Отключать `deadlock_detection`, `dhat-heap`, `spawn_chunk_display` и `slow_chunk_gen`.
- Включать `stand-alone` только когда хотите, чтобы серверный бинарный файл предоставлял файл иконки по умолчанию.
- Включать `jaeger` только при наличии настроенного сборщика OTLP трассировки.
- Использовать `fmt` и `minecraft-src` для разработки и проверки сгенерированного кода, но не для обычных продакшен-сборок.
