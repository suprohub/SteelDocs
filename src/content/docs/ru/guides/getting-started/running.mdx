Это руководство описывает запуск Steel и его поведение.

## Запуск сервера

```bash
# Из каталога SteelMC
cargo run --release

# Или запустить готовый бинарный файл
./steel
```

При запуске Steel:

1. Загрузит или создаст `config/config.toml`
2. Проверит конфигурацию
3. Инициализирует два асинхронных рантайма
4. Начнёт слушать настроенный порт (по умолчанию: 25565)

## Подключение

Используйте Minecraft Java Edition **{VERSION}** для подключения:

1. Откройте Minecraft и перейдите в Сетевую игру
2. Добавьте сервер с адресом `localhost` (или IP вашего сервера)
3. Подключитесь и играйте

## Корректное завершение

Нажмите `Ctrl+C`, чтобы остановить сервер. Steel:

1. Прекратит принимать новые подключения
2. Сохранит все изменённые чанки на диск
3. Корректно завершит работу

Вы увидите вывод, похожий на:

```
Saved 42 chunks
```

## Данные мира

Steel хранит данные мира в текущем каталоге:

- `config/` - файлы конфигурации
- `saves/` - данные сохранения мира (чанки в формате NBT)

## Состояние сервера

Список игроков отображает метрики производительности в реальном времени:

- **TPS** - тиков в секунду (цель: 20)
- **MSPT** - миллисекунд на тик (чем меньше, тем лучше)

## Режимы тикрейта

Steel поддерживает различные режимы тикрейта для отладки:

- **Normal** - стандартные 20 TPS
- **Frozen** - приостановлено (тики не выполняются)
- **Sprint** - максимальная скорость

## Устранение неполадок

### Порт уже используется

Если вы видите ошибку привязки, другой процесс использует порт 25565. Либо:

- Остановите другой процесс
- Измените `server_port` в конфигурации

### Клиент не может подключиться

Убедитесь, что:

- Клиент работает с Minecraft {VERSION}
- `online_mode` соответствует вашей настройке (установите `false` для оффлайн/пиратских клиентов)
- Брандмауэр разрешает подключения на настроенном порту

### Ошибки конфигурации

Если Steel немедленно завершает работу, проверьте сообщение об ошибке. Частые проблемы:

- `simulation_distance` больше, чем `view_distance`
- `compression.threshold` меньше 256
- `enforce_secure_chat` включён без `online_mode` и `encryption`
