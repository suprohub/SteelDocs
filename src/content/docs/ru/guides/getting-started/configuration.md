Steel использует конфигурационный файл в формате TOML, который поддерживает комментарии.

## Файл конфигурации

При первом запуске Steel создаёт `config/config.toml` со значениями по умолчанию. Ниже приведён минимальный рабочий вариант конфигурации. Эта документация поможет вам запустить Steel; для более глубокого погружения во все возможности обратитесь к [этому руководству](../configuration/overview).

## Полный пример

```toml
[server]
# Порт сервера
server_port = 25565
# Максимальное количество игроков на сервере
max_players = 20
# Максимальная дальность обзора в чанках
view_distance = 10
# Максимальная дистанция симуляции в чанках
simulation_distance = 10
# Использовать ли службу аутентификации Mojang
online_mode = true
# Включить ли шифрование связи клиент-сервер
encryption = true
# Сообщение дня, отображаемое в списках серверов
motd = "A Steel Server"
# Использовать ли пользовательскую иконку сервера
use_favicon = true
# Путь к файлу иконки (формат PNG, 64x64 пикселя)
favicon = "config/favicon.png"
# Принудительно использовать безопасный чат
enforce_secure_chat = false

# Настройки сжатия
[server.compression]
threshold = 256
level = 4
```

## Правила валидации

Steel проверяет конфигурацию при запуске:

| Параметр                 | Ограничение                                           |
| ------------------------ | ----------------------------------------------------- |
| `server_port`            | 1–65000                                               |
| `view_distance`          | 1–32                                                  |
| `simulation_distance`    | 1–32, должно быть ≤ `view_distance`                   |
| `compression.threshold`  | ≥ 256                                                 |
| `compression.level`      | 0–9                                                   |
| `enforce_secure_chat`    | Требует, чтобы `online_mode` и `encryption` были `true` |

Если валидация не пройдена, сервер завершит работу с сообщением об ошибке.
