Это руководство описывает, как собрать и запустить Steel.

## Требования

- **Rust nightly toolchain** - Steel использует возможности редакции Rust 2024
- **64-битная операционная система** - Linux, macOS или Windows
- **Git** - для клонирования репозитория

## Установка Rust

Если у вас не установлен Rust, используйте [rustup](https://rustup.rs/):

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

Steel требует инструментарий nightly. Репозиторий включает `rust-toolchain.toml`, который автоматически выбирает правильную версию.

## Сборка из исходников

```bash
# Клонируйте репозиторий
git clone https://github.com/Steel-Foundation/SteelMC.git
cd SteelMC

# Соберите в релизном режиме (рекомендуется для запуска)
cargo build --release

# Бинарный файл будет находиться в target/release/steel
```

### Сборочные команды

| Команда                 | Назначение                                      |
| ----------------------- | ----------------------------------------------- |
| `cargo build`           | Отладочная сборка (быстрее компиляция, медленнее выполнение) |
| `cargo build --release` | Релизная сборка (медленнее компиляция, оптимизировано)    |
| `cargo check`           | Быстрая проверка синтаксиса и типов             |
| `cargo test`            | Запуск тестов                                   |
| `cargo clippy`          | Запуск линтера                                  |

### Сборочные функции

Steel поддерживает опциональные функции сборки, которые можно включить с помощью `--features`:

```bash
# Включить обнаружение взаимных блокировок (только для отладки)
cargo build --features deadlock_detection

# Включить профилирование кучи с dhat
cargo build --features dhat-heap

# Отключить стандартный аллокатор mimalloc
cargo build --no-default-features
```

**Доступные функции:**

| Функция              | Описание                                                     |
| -------------------- | ------------------------------------------------------------ |
| `mimalloc`           | Использовать аллокатор mimalloc (включён по умолчанию)       |
| `deadlock_detection` | Включить обнаружение взаимных блокировок parking_lot для отладки проблем с блокировками |
| `dhat-heap`          | Включить профилирование кучи с dhat                          |

**Обнаружение взаимных блокировок** особенно полезно при разработке, если вы сталкиваетесь с зависаниями или подозреваете проблемы с блокировками. При включении parking_lot будет обнаруживать потенциальные взаимные блокировки и вызывать панику с диагностической информацией.

## Запуск сервера

```bash
# Запустить напрямую через cargo (отладочный режим)
cargo run

# Запустить с релизными оптимизациями
cargo run --release

# Или запустить собранный бинарный файл напрямую
./target/release/steel
```

Сервер:

1. Создаст файл `config/steel_config.json5`, если он не существует
2. Привяжется к `0.0.0.0:25565` по умолчанию
3. Начнёт принимать клиенты Minecraft версии {await mcVersion()}

## Готовые бинарные сборки

Готовые бинарные файлы доступны на странице [GitHub Releases](https://github.com/Steel-Foundation/SteelMC/releases) для:

- Linux (x86_64)
- Windows (x86_64)
- macOS (x86_64 и ARM)
