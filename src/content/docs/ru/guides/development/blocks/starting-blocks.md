> ⚠️ Это лишь самая базовая настройка и **пока не добавляет никакой функциональности**.

---

## 1. Выберите блок для добавления

Сначала выберите, какой блок вы хотите добавить в проект.

**Пример:** В этом руководстве мы хотим добавить **железные прутья** и **медные прутья**.

---

## 2. Проверьте имя класса в `classes.json`

Прежде чем создавать структуру, нужно правильно её назвать.

Откройте файл:

```
steel-core/build/classes.json
```

Найдите в нём ваш блок. В нашем примере:
- Находим `IronBarsBlock`
- Находим `WeatheringCopperBarsBlock`

Это означает, что для управления обоими блоками нужны **две разные структуры**.

---

## 3. Создайте файл класса блока

Создайте класс в:

```
steel-core/src/behavior/blocks/
```

Название файла должно быть **максимально точным**. Для нашего примера:
- `iron_bars_block.rs`
- `copper_bars_block.rs`

---

## 4. Добавьте определение структуры

Добавьте в файл структуру следующим образом:

```rust
// /steel-core/src/behavior/blocks/iron_bars_block.rs
pub struct IronBarsBlock {
    block: BlockRef,
}

impl IronBarsBlock {
    /// Создаёт новое поведение для блока-прутьев.
    #[must_use]
    pub const fn new(block: BlockRef) -> Self {
        Self { block }
    }
}

impl BlockBehaviour for IronBarsBlock {}
```

> ⚠️ Это лишь базовая настройка и **пока не даёт никакой функциональности!**

---

## 5. Зарегистрируйте модуль блока
Чтобы зарегистрировать блок, необходимо добавить атрибут `block_behavior`!
```rust
// /steel-core/src/behavior/blocks/iron_bars_block.rs
#[block_behavior]
pub struct IronBarsBlock {
    block: BlockRef,
}

impl IronBarsBlock {
    /// Создаёт новое поведение для блока-прутьев.
    #[must_use]
    pub const fn new(block: BlockRef) -> Self {
        Self { block }
    }
}

impl BlockBehaviour for IronBarsBlock {}
```

> ⚠️ Более сложные блоки, чем железные прутья, имеют свойства. Дополнительная информация [здесь](../block_item_registration)!

---

## 6. Скомпилируйте проект

Теперь нажмите **компилировать** и позвольте Rust (и нашей конфигурации) совершить магию!

После компиляции ваш блок должен появиться в:

```
steel-core/src/behavior/generated/blocks.rs
```

Перейдите туда и используйте **Ctrl + F**, чтобы найти имя вашего блока.

### Устранение неполадок

Если блок всё ещё отсутствует:

1. Удалите папку `generated`
2. Выполните:

   ```
   cargo clean
   ```
3. Скомпилируйте снова

Это должно решить проблему.

---

# Добавление поведения блоку

Как уже было сказано, сейчас блок **ничего не делает**.

Чтобы добавить поведение, реализуйте необходимые методы трейта `BlockBehaviour` в вашем файле (например, `iron_bars_block.rs`).

👉 **Я рекомендую** посмотреть на реализации других блоков со схожей функциональностью.

Для лучшего понимания вот некоторая информация:

---

## Работа с состояниями блока

### Получение состояния блока

Чтобы получить состояние блока, можно сделать так:

```rust
let west_pos = Direction::West.relative(pos);
let west_state = world.get_block_state(&west_pos);
```

В этом состоянии блока сохранена **вся информация** о конкретном блоке.

---

### Изменение свойств состояния блока

Изменить можно так:

```rust
state.set_value(&BlockStateProperties::WEST, true);
```

Получение значения - аналогично.

---

## Проверка соседних блоков или тегов

Чтобы проверить, является ли соседний блок или группа блоков определённым блоком или группой (например, прутья или стены), используйте:

```rust
let walls_tag = Identifier::vanilla_static("walls");
if REGISTRY.blocks.is_in_tag(neighbor_block, &walls_tag) {
    return true;
}
```

---

Вот и всё - у вас есть **базовая структура** и можно начинать реализовывать настоящее поведение 🚀
