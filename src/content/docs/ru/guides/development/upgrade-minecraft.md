Это руководство объясняет всё, что нужно для обновления версии Minecraft в Steel.
Также даны ссылки для углублённого изучения.

Как можно догадаться, обновление проекта Steel на новую версию Minecraft - задача не из лёгких, так что не торопитесь!

Какие знания потребуются?

- Java (без шуток) и Rust
- Gradle, будет полезен

Чтобы обновить Steel, нам нужна вся информация от Minecraft для новой версии. Это означает, что процесс обновления начинается с SteelExtractor.
Если вы не знаете, что это такое, вся необходимая информация находится [здесь](../tools/steel_extractor).

Для плавного перехода и минимизации рабочей нагрузки рекомендуется начинать обновление, пока доступны только снапшот-версии.

## 1. Изменение версии Minecraft в SteelExtractor

Версия Minecraft и все связанные версии зависимостей настраиваются в `gradle.properties`:

```properties
minecraft_version=26.1-snapshot-11
loader_version=0.18.4
loom_version=1.15-SNAPSHOT
fabric_kotlin_version=1.13.9+kotlin.2.3.10
```

Чтобы обновиться на новую версию Minecraft:

1. Измените `minecraft_version` на целевую версию
2. Обновите `fabric_version` до совместимой версии Fabric API для этой версии Minecraft
3. При необходимости обновите `loader_version` и `fabric_kotlin_version`
4. Если применимо, обновите версию маппингов Parchment в `build.gradle`

Правильные версии можно найти на [https://fabricmc.net/develop](https://fabricmc.net/develop).

Затем запустите Minecraft и извлеките данные, как описано [здесь](../tools/steel_extractor/).
Для снапшотов 26.1 потребовалось более существенно изменить систему Gradle и обновить Java 21 до Java 25.
Все файлы, затронутые для этого снапшота, можно найти здесь: https://github.com/JunkyDeveloper/SteelExtractor/commit/992ae692f8dcab02edba96308d30422f43f1961e

## 2. Копирование JSON-файлов SteelExtractor

Переместите все JSON-файлы в соответствующие места. В новейшей версии в рабочем каталоге уже есть папки, которые один в один соответствуют папкам в Steel.

## 3. Извлечение данных из Minecraft

Получите jar-файл новейшей версии Minecraft. Это можно сделать через лаунчер или официальную загрузку от Mojang. После загрузки jar-архив нужно извлечь.
При этом откроется следующая структура папок:

```
└── 📁minecraft-26.1-snapshot-11-client
    └── 📁assets
    └── 📁com
    └── 📁data
    └── 📁META-INF
    └── 📁net
    ├── flightrecorder-config.jfc
    ├── pack.png
    └── version.json
```

Теперь скопируйте папку `data` в: `steel-registry/build_assets/builtin_datapacks/minecraft/data`
Да, вторая папка `minecraft` - это та же самая папка. Таким образом, структура:

```
└── 📁build_assets
    └── 📁builtin_datapacks
        └── 📁minecraft
            └── 📁data
                └── 📁minecraft
                    └── 📁advancement
                    └── 📁equipment
                    └── 📁font
                    └── 📁frog_variant
                    └── 📁instrument
                    └── 📁items
                    ...
                    └── 📁zombie_nautilus_variant
                ├── .mcassetsroot
            ├── pack.mcmeta
    ├── attributes.json
    ├── block_entities.json
    ├── blocks.json
    ├── entities.json
    ├── fluids.json
    ├── game_rules.json
    ├── items.json
    ├── level_events.json
    ├── menutypes.json
    ├── mob_effects.json
    ├── multi_noise_biome_source_parameters.json
    ├── packets.json
    ├── potions.json
    ├── sound_events.json
    ├── sound_types.json
    └── tags.json
```

## 4. Восстановление реестра

Процесс восстановления делится на два этапа.
В большинстве случаев реестр сломан, что не даёт Steel собраться, потому что некоторые структуры больше не совпадают. Поэтому начнём с исправления реестра.

Реестр Steel состоит из трёх папок.
Одна из них - `build_assets`, с которой мы уже работали.
Другая - `build`, содержащая файлы Rust, которые генерируют другие файлы Rust на основе содержимого JSON-файлов. Здесь происходит первый этап.
Рекомендуется выполнить `git diff` после копирования новых файлов, чтобы увидеть, какие файлы изменились и как изменилась структура. Это поможет определить, какие реестры нужно обновить.
После этого внесите необходимые изменения в каталог `build`. Затем вы, вероятно, снова столкнётесь с ошибками компиляции, но на этот раз в самих реестрах, поскольку они ещё не обновлены.
Примените те же структурные изменения и там. После этого всё должно скомпилироваться.

## 5. Восстановление остального

Этот шаг включает исправление оставшегося кода и реализацию новых функций, появившихся в новой версии. Конкретная работа зависит от конкретной версии, так что удачного обновления!
