> ⚠️ Это лишь самая базовая настройка и **пока не добавляет никакой функциональности**.

---

## 1. Выберите предмет для добавления

Сначала выберите, какой предмет вы хотите добавить в проект.

**Пример:** В этом руководстве мы хотим добавить предмет `Shovel` (лопата).

---

## 2. Проверьте имя класса в `classes.json`

Прежде чем создавать структуру, нужно правильно её назвать.

Откройте файл:

```
steel-core/build/classes.json
```

Найдите строки, похожие на имя предмета, который вы хотите реализовать. В нашем примере:
- Находим `ShovelItem`

Это означает, что для управления обоими предметами нужны **две разные структуры** (в данном случае только одна, но принцип общий).

---

## 3. Создайте файл класса предмета

Создайте класс в:

```
steel-core/src/behavior/items/
```
TODO: Если вы читаете это, пожалуйста, уточните в #terminology, зафиксировано ли что-то
Название файла должно быть **максимально точным**. Для нашего примера:
- `shovel.rs`

---

## 4. Добавьте определение структуры

Добавьте в файл структуру следующим образом:

```rust
// /steel-core/src/behavior/items/shovel.rs
pub struct ShovelBehavior;

impl ItemBehavior for ShovelBehavior {}
```

> ⚠️ Это лишь базовая настройка и **пока не даёт никакой функциональности!**

---

## 5. Зарегистрируйте модуль предмета
Чтобы зарегистрировать предмет, необходимо добавить атрибут `item_behavior`!

```rust
// /steel-core/src/behavior/items/shovel.rs
#[item_behavior(class = "ShovelItem")]
pub struct ShovelBehavior;

impl ItemBehavior for ShovelBehavior {}
```

> ⚠️ Более сложные предметы, чем лопата, имеют свойства. Дополнительная информация [здесь](../block_item_registration)!

---

## 6. Скомпилируйте проект

Теперь нажмите **компилировать** и позвольте Rust (и нашей конфигурации) совершить магию!

После компиляции ваш предмет должен появиться в:

```
steel-core/src/behavior/generated/items.rs
```

Перейдите туда и используйте **Ctrl + F**, чтобы найти имя вашего предмета.

### Устранение неполадок

Если предмет всё ещё отсутствует:

1. Удалите папку `generated`
2. Выполните:

   ```
   cargo clean
   ```
3. Скомпилируйте снова

Это должно решить проблему.

---

# Добавление поведения предмету

Как уже было сказано, сейчас предмет **ничего не делает**.

Чтобы добавить поведение, реализуйте необходимые методы трейта `ItemBehavior` в вашем файле (например, `shovel.rs`).

👉 **Я рекомендую** посмотреть на реализации других предметов со схожей функциональностью.
